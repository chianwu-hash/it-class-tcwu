# 重構前測試規格

更新日期：2026-03-18

本文件用來固定目前網站在重構前應維持不變的功能行為。後續每一階段重構完成後，都必須依這份規格驗證，避免「結構變乾淨，但功能默默壞掉」。

## 測試原則

- 先測本地，再測遠端
- 每次只驗證本次改動相關範圍
- 測試結果必須記錄：通過 / 不通過 / 待確認
- 若重構不涉及某模組，不需要每次全站回歸，但核心登入流程要至少抽測一次

## 測試環境

### 本地

- Windows 11
- Python 3 本機靜態 server 或其他本機 server
- 建議使用：`http://localhost:3000`

### 遠端

- Netlify production
- Vercel production

### 後端

- Supabase production project
- Google OAuth / Supabase Auth 已配置可用

## 核心回歸清單

### A. 全站入口與導航

#### A-1 全站首頁

頁面：`/index.html`

檢查：

- 首頁可正常載入
- 三年級入口可進入 `grade3/index.html`
- 六年級入口可進入 `grade6/index.html`
- 已登入時，教師帳號可看到後台入口
- 未登入時，不顯示教師專用入口

#### A-2 年級首頁導覽列

頁面：

- `grade3/index.html`
- `grade6/index.html`

檢查：

- 導覽列正常顯示
- 週次導航正確
- 隱藏的週次不顯示於導覽列
- 已顯示的週次可正確進入對應頁面

### B. 週卡片顯示/隱藏

#### B-1 教師後台切換週卡片

頁面：`/admin-progress.html`

檢查：

- 教師登入後可看到週卡片顯示管理
- 點擊 `顯示/隱藏` 可成功寫入資料庫
- 切換後按鈕狀態會更新

#### B-2 年級首頁同步隱藏

頁面：

- `grade3/index.html`
- `grade6/index.html`

檢查：

- 被隱藏的週卡片不顯示
- 被隱藏的週次導覽也不顯示
- 本地與遠端結果一致
- `.html` 與 Pretty URL 都能正確判斷週次

### C. 學生登入與 session

#### C-1 首頁登入

頁面：`/index.html`

檢查：

- 點登入可成功走 Google OAuth
- 登入後 session 可被週頁讀到
- 教師帳號登入後能看到後台入口

#### C-2 三年級週頁登入

頁面：

- `grade3/week04.html`
- `grade3/week05.html`
- `grade3/week06.html`

檢查：

- 未登入時不能開始打字闖關，必須先登入再練習並保存進度
- 登入後會顯示保存狀態
- 登入後可正常讀取既有進度

#### C-3 教師後台登入

頁面：`/admin-progress.html`

檢查：

- 教師登入後顯示 email
- 教師能查詢學生進度與操作按鈕
- 切去別頁再回來，session 不應消失
- 返回頁面後按鈕仍可操作

### D. 三年級 week04 闖關

頁面：`grade3/week04.html`

檢查：

- 頁面正常顯示
- 關卡 1~5 可依序解鎖
- 注音切換可用
- 過關後顯示對應鼓勵語
- 過關後能保存進度
- 重新整理後能續關
- 全破後可重新闖關
- 通關後再按檢查答案會重播成就效果，不顯示錯誤

### E. 三年級 week05 闖關

頁面：`grade3/week05.html`

檢查：

- 關卡內容為正式文案
- 第 4 關有換行
- 關卡 1~5 可依序解鎖
- 注音切換可用
- 阿德勒式鼓勵語依關卡不同
- 過關後保存進度成功
- 最後一關動畫與 `completed=true` 寫入同步
- 全破後重新闖關可正常重設
- 通關後再按檢查答案會重播成就效果
- 進度寫入失敗時會顯示除錯資訊

### F. 三年級 week06 任務頁

頁面：`grade3/week06.html`

檢查：

#### F-1 任務一：資安測驗

- 題目圖片正常顯示
- 選項可點選
- 提交後可顯示分數
- 已登入時分數可寫入 `score`

#### F-2 任務二：Email 任務

- 主旨格式顯示為 `班級座號姓名`
- 範例顯示 `30108 王小明`
- 任務說明文字正確

#### F-3 任務三：中打闖關

- 關卡 1~5 可依序解鎖
- 第 2 關起有換行
- 第 2、3、5 關可輸入換行
- 第 4 關題目字級正常偏大
- 過關鼓勵語採阿德勒式語氣
- 答錯時可提示錯誤位置類型
- 過關後可保存進度並續關

### G. 教師後台資料查詢

頁面：`/admin-progress.html`

檢查：

- 週次篩選有效
- 活動類型篩選有效
- 完成狀態篩選有效
- email / activity_key 搜尋有效
- 表格欄位顯示正確：
  - 學生
  - 週次
  - 活動
  - 類型
  - 目前進度
  - 分數
  - 狀態
  - 最後更新
  - 操作
- `重設` 可清除指定學生指定活動進度

## 建議測試紀錄格式

每次實作完成後，用這種格式記錄：

- 測試日期：
- 分支：
- 影響範圍：
- 本地測試：通過 / 失敗
- 遠端測試：通過 / 失敗
- 備註：

範例：

- 測試日期：2026-03-18
- 分支：`refactor/site-foundation`
- 影響範圍：shared auth
- 本地測試：A-1、C-1、C-2(week05)、C-3 通過
- 遠端測試：A-1、C-1、C-3 通過
- 備註：Firefox 返回頁後 session 恢復需特別驗證

## 目前重構前風險提醒

- PowerShell 顯示亂碼不代表瀏覽器真的亂碼
- `week06.html` 屬於高複雜頁面，應放在 `week04` / `week05` 之後重構
- `admin-progress.html` 需要特別測試瀏覽器返回頁、切分頁、重新聚焦等情境
- 本地與遠端都要驗證首頁週卡顯示，避免 Pretty URL 差異再次回歸
